## 一、当前整体架构

```
              墨现 Guida           遁甲 Dunjia           UMI 简智新创
         (MKV + JSONL + CSV)      (.mcap 文件)         (.mcap 文件, 双robot)
                  │                    │                     │
                  ▼                    ▼                     ▼
         guida_reader.py       dunjia_reader.py       umi_reader.py
                  │                    │                     │
                  └────────────────────┼─────────────────────┘
                                       │
                                       ▼
                            zpds_prepare/main.py
                         --profile guida/dunjia/umi
                         5 项检测 → 分段方案
                         ① 帧数一致性  ② 坏帧
                         ③ 黑屏区间    ④ 视频缺口
                         ⑤ IMU 缺口
                                       │
                                       ▼
                     output/{moxian,dunjia,umi}/
                     ├── quality_issues.json
                     └── segment_candidates.json
                                       │
                                       ▼
                            batch_prepare.py
                       --profile guida/dunjia/umi
                       批量生成 Prepared Segment
                                       │
                                       ▼
              prepared_segments/{moxian,dunjia,umi}/
              ├── batch_summary.json
              ├── seg_000001/
              ├── seg_000002/
              └── ...
```

**核心设计**：5 个检测器 + 2 个决策器 + 2 个写入器 + validator 全部零改动复用，仅通过 reader 适配不同数据源。reader 统一返回 Session 对象，调用方遍历 `video_streams` / `imu_streams` 字典，文件名由 `stream_id` 动态生成。

## 二、命令

### 命令 1：异常检测 → 分段方案

```bash
# 墨现 (默认 profile)
python -m zpds_prepare.main "E:/datasets/egos/墨现" --output output/moxian/

# 遁甲
python -m zpds_prepare.main /path/to/session.mcap --profile dunjia --output output/dunjia/

# UMI
python -m zpds_prepare.main /path/to/01767.mcap --profile umi --output output/umi/
```

**6 步管线**：

| 步骤 | 检测项 | 模块 | issue_type |
|------|--------|------|-----------|
| 1 | 读取原始数据（按 profile 分发 reader，统一返回 Session） | `readers/guida_reader.py` / `readers/dunjia_reader.py` / `readers/umi_reader.py` | — |
| 2a | 帧数一致性 | `detectors/frame_count.py` | `frame_count_mismatch` |
| 2b | 坏帧检测 | `detectors/bad_frame.py` | `bad_frame` |
| 2c | 黑屏区间 | `detectors/black_frame.py` | `continuous_black_frames` |
| 2d | 视频时间戳缺口 | `detectors/timestamp_gap.py` | `timestamp_gap` |
| 2e | IMU 时间戳缺口 | `detectors/imu_gap.py` | `imu_gap` |
| 3 | 汇总分析 | `decisions/segment_planner.py` | — |
| 4 | 写出 quality_issues.json | `writers/quality_writer.py` | — |
| 5 | 生成候选 Segment | `decisions/segment_planner.py` | — |
| 6 | 写出 segment_candidates.json | `writers/candidate_writer.py` | — |

### 命令 2：批量生成 Prepared Segment

```bash
# 墨现 (默认)
python batch_prepare.py --profile guida

# 遁甲
python batch_prepare.py --candidates output/dunjia/segment_candidates.json --profile dunjia

# UMI
python batch_prepare.py --profile umi --dataset /path/to/01767.mcap -c output/umi/segment_candidates.json -o prepared_segments/umi/
```

### 3.4 Profile 分发机制

```python
def _get_reader(profile: str):
    """所有 reader 模块导出统一的 read_session() 入口"""
    if profile == "guida":
        from zpds_prepare.readers import guida_reader as rd
    elif profile == "dunjia":
        from zpds_prepare.readers import dunjia_reader as rd
    elif profile == "umi":
        from zpds_prepare.readers import umi_reader as rd
    return rd
```

所有 reader 统一返回 `Session` 对象（含 `video_streams` / `imu_streams` 字典），调用方遍历流处理，不再硬编码文件名。

| Profile | 视频 | IMU | 流 ID | 畸变模型 |
|---------|------|-----|-------|---------|
| `guida` | MKV MJPG, 30fps | CSV, 50Hz | `ego_rgb`, `ego_imu` | — (从 meta.json) |
| `dunjia` | CompressedVideo H264, 25fps | foxglove.Imu, 196Hz | `camera0/1/2`, `robot0_imu` | `plumb_bob` |
| `umi` | CompressedImage H264, 30fps | IMUMeasurement, ~195Hz | `robot0/1_camera0`, `robot0/1_imu` | `equidistant` + `T_b_c` |

## 四、统一异常模型：QualityIssue

所有 5 个检测器的输出统一为此数据类：

```python
@dataclass
class QualityIssue:
    issue_type: str              # frame_count_mismatch | bad_frame | continuous_black_frames | timestamp_gap | imu_gap
    stream_id: str               # ego_rgb | ego_imu
    start_ns: int                # 异常起始 (设备时钟)
    end_ns: int                  # 异常结束 (设备时钟)
    severity: str                # warning | error | critical
    decision: str                # trim | split | keep_with_flag
    details: dict                # 检测器附加信息
```

### 5 种 issue_type 速查

| issue_type | 检测器 | 典型 decision |
|-----------|--------|--------------|
| `frame_count_mismatch` | `frame_count.py` | `keep_with_flag`（差异≤5 → warning，>5 → error） |
| `bad_frame` | `bad_frame.py` | `keep_with_flag`（标记，不切分） |
| `continuous_black_frames` | `black_frame.py` | 首尾 → `trim`，中间长 → `split` |
| `timestamp_gap` | `timestamp_gap.py` | ≥0.5s → `split`，<0.5s → `keep_with_flag` |
| `imu_gap` | `imu_gap.py` | ≥1.0s → `split`，<1.0s → `keep_with_flag` |

## 五、候选 Segment 生成算法

```
输入: issues (list[QualityIssue]), session_start_ns, session_end_ns

1. 分离 issue:
   - decision = "trim" 且靠近起始 → head_trims
   - decision = "trim" 且靠近末尾 → tail_trims
   - decision = "split" → split_boundaries

2. 计算有效范围:
   valid_start = max(session_start, 最晚 head_trim.end)
   valid_end   = min(session_end,   最早 tail_trim.start)

3. 切分:
   按 split_boundaries 中的缺口切割
   [valid_start, gap1.start], [gap1.end, gap2.start], ...

4. 过滤:
   - 丢弃 duration < min_duration_s (默认 1s)
   - 标记 duration > max_duration_s (默认 120s)
```

## 六、Prepared Segment 产出结构

```
prepared_segments/{moxian,dunjia}/seg_000001/
├── segment.json                    # 核心控制文件
│   ├── segment_id, source_session
│   ├── source_assets (源文件 SHA-256)
│   ├── timeline (相对时钟: 0 → duration_ns)
│   ├── source_span (设备时钟范围)
│   ├── streams[ego_rgb + ego_imu]
│   ├── calibration_uri
│   └── quality (status + issues，从 candidates 传入)
├── data/
│   ├── ego_rgb.mp4                 # H.264 CFR 转码视频
│   └── imu.parquet                 # 相对时间 + 标准列名
├── maps/
│   └── rgb_sample_map.parquet      # 输出帧↔源帧 最近邻映射
├── calibration/
│   └── calibration.json            # 针孔模型内参 + 外参
└── reports/
    └── validation.json             # 6项自动校验
```
