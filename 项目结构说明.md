# ZPDS — 具身智能数据标准化管线

## 项目简介

ZPDS 是具身智能数据的标准化处理管线，目标是统一多个采集源（墨现、遁甲、简智新创、A2D 真机、EPIC-KITCHENS-100）的数据格式，经过质量检查、场景分割、标注后，产出可直接用于训练的标准 Prepared Segment。

## Profile 双数据源

main.py 和 batch_prepare.py 通过 `--profile` 参数支持两种数据源，**5 个检测器 + 2 个决策器 + 2 个写入器 + validator 全部零改动复用**：

| Profile | 数据源 | 容器格式 | 视频 | IMU | 标定来源 |
|---------|--------|---------|------|-----|---------|
| `guida` (默认) | 墨现 | 目录 (MKV + JSONL + CSV) | color_000000.mkv, 30fps | imu_000000.csv, 50Hz | meta.json |
| `dunjia` | 遁甲 | MCAP (Foxglove protobuf) | H264 压缩视频, 25fps | Protobuf IMU, 196Hz | CameraCalibration topic |

## zpds_prepare/ — 异常检测 → 分段方案

将"检测到异常"变成"可执行的分段方案"。5 个检测器输出统一的 `QualityIssue`，
再由 `segment_planner` 生成候选 Segment。

### 命令

```bash
# 墨现 (默认)
python -m zpds_prepare.main "E:/datasets/egos/墨现" --output output/moxian/

# 遁甲
python -m zpds_prepare.main /path/to/session.mcap --profile dunjia --output output/dunjia/
```

### 八步管线

| 步骤 | 内容 | 负责模块 |
|------|------|----------|
| ① 读取原始数据 | 根据 profile 选择 reader | `readers/guida_reader.py` 或 `readers/dunjia_reader.py` |
| ②a 帧数一致性 | 对比 index 声明帧数 vs meta 声明帧数 | `detectors/frame_count.py` |
| ②b 坏帧检测 | 扫描视频文件检测解码失败帧 | `detectors/bad_frame.py` |
| ②c 黑屏区间 | 逐帧灰度均值检测 + 连续区间合并 + trim/split 决策 | `detectors/black_frame.py` |
| ②d 视频缺口 | 相邻帧间隔异常检测 + 分级处置 | `detectors/timestamp_gap.py` |
| ②e IMU 缺口 | IMU 采样间隔异常检测 + 分级处置 | `detectors/imu_gap.py` |
| ③ 汇总分析 | 按类型 / 按处置统计 Issue | `decisions/segment_planner.py` |
| ④ 写出质量报告 | quality_issues.json | `writers/quality_writer.py` |
| ⑤ 生成候选 Segment | 根据 trim/split 决策切分 Session | `decisions/segment_planner.py` |
| ⑥ 写出分段方案 | segment_candidates.json | `writers/candidate_writer.py` |

### 统一异常记录格式

```python
@dataclass
class QualityIssue:
    issue_type: str      # continuous_black_frames | timestamp_gap | imu_gap | ...
    stream_id: str       # ego_rgb | ego_imu
    start_ns: int        # 异常起始 (设备时钟)
    end_ns: int          # 异常结束 (设备时钟)
    severity: str        # warning | error | critical
    decision: str        # trim | split | keep_with_flag | quarantine
    details: dict        # 附加信息 (帧数、间隔、阈值等)
```

### 处置决策

| 位置 | 决策 | 说明 |
|------|------|------|
| 开头连续黑屏 | `trim` | 裁剪掉，不计入有效范围 |
| 结尾连续黑屏 | `trim` | 裁剪掉，不计入有效范围 |
| 中间长黑屏 | `split` | 在此处切分 |
| 中间短黑屏 | `keep_with_flag` | 保留并标记 |
| 视频小缺口 (<0.5s) | `keep_with_flag` | 保留并标记 |
| 视频大缺口 (≥0.5s) | `split` | 在此处切分 |
| IMU 小缺口 (<1.0s) | `keep_with_flag` | 保留并标记 |
| IMU 大缺口 (≥1.0s) | `split` | 在此处切分或隔离 |

### 产出结构

```
output/{moxian,dunjia}/
├── quality_issues.json       # 所有异常的统一记录
└── segment_candidates.json   # 候选 Segment 列表
```

## segment/ — Prepared Segment 生成模块

### 七步管线

| 步骤 | 输入 | 输出 | 负责模块 |
|------|------|------|----------|
| ① 确定有效区间 | index 索引 + QC 结果 | source_start_ns / source_end_ns | `span_determiner.py` |
| ② 视频裁剪转码 | 源视频 + Span | data/ego_rgb.mp4 (H.264 CFR) | `video_transcoder.py` |
| ③ 采样映射表 | index 索引 + Span | maps/rgb_sample_map.parquet | `sample_map.py` |
| ④ IMU 规范化 | IMU 原始数据 + Span | data/imu.parquet | `imu_normalizer.py` |
| ⑤ 标定提取 | meta.json / MCAP calibration | calibration/calibration.json | `calibration.py` |
| ⑥ 控制文件 | 全部上游结果 | segment.json | `segment_writer.py` |
| ⑦ 写出后验证 | 全部输出文件 | reports/validation.json | `validator.py` |

### 产出结构

```
prepared_segments/{moxian,dunjia}/seg_000001/
├── segment.json                    # 核心控制文件（streams/timeline/quality）
├── data/
│   ├── ego_rgb.mp4                 # H.264 CFR 转码视频
│   └── imu.parquet                 # 规范化 IMU（相对时间 + 标准列名）
├── maps/
│   └── rgb_sample_map.parquet      # 输出帧 ↔ 源帧映射（最近邻）
├── calibration/
│   └── calibration.json            # RGB 内参 + IMU/深度 外参
└── reports/
    └── validation.json             # 6 项自动校验
```

### write-validate 闭环

每次写出后自动验证 6 项：

| 检查项 | 说明 |
|--------|------|
| 文件引用完整性 | segment.json 中引用的所有文件是否真实存在 |
| 视频可解码 | ego_rgb.mp4 能否被 OpenCV 正常打开 |
| 帧数一致 | 视频帧数 == sample_map 行数 |
| 映射时间单调 | output_timestamp_ns 严格递增 |
| IMU 时间单调 | timestamp_ns 非递减 |
| IMU 起点 | 相对时间从 0 附近开始 |

## batch_prepare.py — 批量 Prepared Segment 生成

读取 segment_candidates.json，对每个候选区间自动生成完整 Prepared Segment。

```bash
# 墨现 (默认)
python batch_prepare.py --profile guida

# 遁甲
python batch_prepare.py --candidates output/dunjia/segment_candidates.json --profile dunjia
```

### 工作流程

```
segment_candidates.json
        │
        ▼
  预加载共享资源（index 索引 + calibration）
        │
        ├── candidate_000001 → seg_000001/
        ├── candidate_000002 → seg_000002/
        └── candidate_000003 → seg_000003/
                    │
                    ▼
          prepared_segments/{moxian,dunjia}/
          ├── batch_summary.json       ← 批量汇总
          ├── seg_000001/              ← 完整 Prepared Segment
          ├── seg_000002/
          └── seg_000003/
```

### 共享优化

- **index 索引** — 全 Session 只读取一次
- **calibration** — 全 Session 只提取一次（标定不随 span 变化）
- 每个 Segment 独立转码视频、生成 sample_map、裁剪 IMU、写入验证

### 与 prepare_segment.py 的区别

| | prepare_segment.py | batch_prepare.py |
|---|---|---|
| 输入 | 硬编码路径 + QC 结果 | segment_candidates.json |
| Span 来源 | `span_determiner.py` 自动判定 | 候选方案文件中指定 |
| Segment 数量 | 始终 1 个 | 按候选方案批量 |
| 共享资源 | 无 | index + calibration 只加载一次 |
| profile 支持 | 无 | ✅ `--profile` |
| 用途 | 单次开发调试 | 批量生产 |

## 配置（config.yaml）

```yaml
# 视频质量检测
video:
  black_threshold: 5
  min_black_duration_s: 0.5
  black_detection:
    mean_intensity_threshold: 5.0
    min_duration_s: 0.5
    edge_tolerance_s: 1.0

# 时间戳异常检测
timestamp:
  video_gap_factor: 2.0
  imu_gap_factor: 3.0
  video:
    gap_factor: 2.0
    split_gap_s: 0.5
  imu:
    gap_factor: 3.0
    split_gap_s: 1.0

# 输出
output:
  target_fps: 30.0
  rgb_codec: "avc1"
  rgb_pix_fmt: "yuv420p"

# Segment 约束
segment:
  min_duration_s: 1.0
  max_duration_s: 120.0
```
