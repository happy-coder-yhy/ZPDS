# ZPDS — 具身智能数据标准化管线

## 项目简介

ZPDS 是具身智能数据的标准化处理管线，目标是统一多个采集源（墨现、遁甲、简智新创、A2D 真机、EPIC-KITCHENS-100）的数据格式，经过质量检查、场景分割、标注后，产出可直接用于训练的标准 Prepared Segment。

## 分支说明

| 分支 | 内容 | 说明 |
|------|------|------|
| `main` | 项目骨架 | 完整的 `zpds/` 包结构、配置、测试框架 |
| `llccxx` | 功能探索 | 墨现数据清洗原型，独立验证后再合入 main |

> 两个分支基于不同提交分叉，文件互不重叠，互不干扰。

## 模块说明

### clean.py — 主程序入口

编排 10 步清洗管线，调用各模块完成检查并汇总结果。

```
python clean.py
```

**检查流程：**

| 步骤 | 检查项 | 负责模块 |
|------|--------|----------|
| 1 | 读取 meta.json | `reader.py` |
| 2 | 统计 index.jsonl 帧数 | `reader.py` |
| 3 | 帧数一致性比较 | `timestamp_checker.py` |
| 4 | 时间戳间隔检测（>40ms 异常） | `timestamp_checker.py` |
| 5 | 视频基本信息（帧数/FPS/分辨率） | `video_checker.py` |
| 6 | 坏帧检测（解码失败） | `video_checker.py` |
| 7 | 黑屏检测（灰度均值 < 5） | `video_checker.py` |
| 8 | 读取 IMU 数据 | `reader.py` |
| 9 | IMU 中断检测（>60ms 异常） | `imu_checker.py` |
| 10 | 生成清洗报告 | `report.py` |

### reader.py — 数据读取

| 函数 | 说明 |
|------|------|
| `read_meta(path)` | 读取 meta.json，提取 device/fps/分辨率/帧数/IMU 采样率 |
| `read_index(path)` | 读取 index.jsonl，提取 type=frame 行的时间戳列表 |
| `read_imu(path)` | 读取 imu_000000.csv，返回 DataFrame |

### timestamp_checker.py — 时间戳检查

| 函数 | 说明 |
|------|------|
| `check_frame_count(index_count, meta_count)` | 比较 index 与 meta 帧数是否一致 |
| `check_timestamp_gaps(timestamps)` | 检测时间戳间隔是否超过阈值（默认 40ms） |

### video_checker.py — 视频检查

| 函数 | 说明 |
|------|------|
| `check_video_info(path)` | 获取帧数/FPS/分辨率 |
| `check_bad_frames(path)` | 统计解码失败的坏帧数 |
| `check_black_frames(path)` | 检测灰度均值 < 5 的黑屏帧 |

### imu_checker.py — IMU 检查

| 函数 | 说明 |
|------|------|
| `check_imu_gaps(imu_df)` | 去重后检测 IMU 时间戳间隔超过 0.06s 的中断 |

### report.py — 报告生成

| 函数 | 说明 |
|------|------|
| `generate_report(results, path)` | 汇总所有检查结果，写入 clean_report.txt，返回 PASS/FAIL |

## prepare_segment.py — Prepared Segment 生成

将墨现 Guida 原始数据转换为标准化 Prepared Segment，一条命令完成：

```
python prepare_segment.py
```

### 七步管线

| 步骤 | 输入 | 输出 | 负责模块 |
|------|------|------|----------|
| ① 确定有效区间 | index.jsonl + QC 结果 | source_start_ns / source_end_ns | `span_determiner.py` |
| ② RGB 裁剪转码 | color_000000.mkv + Span | data/ego_rgb.mp4 (H.264 CFR 30fps) | `video_transcoder.py` |
| ③ 采样映射表 | index.jsonl + Span | maps/rgb_sample_map.parquet | `sample_map.py` |
| ④ IMU 规范化 | imu_000000.csv + Span | data/imu.parquet | `imu_normalizer.py` |
| ⑤ 标定提取 | meta.json | calibration/calibration.json | `calibration.py` |
| ⑥ 控制文件 | 全部上游结果 | segment.json | `segment_writer.py` |
| ⑦ 写出后验证 | 全部输出文件 | reports/validation.json | `validator.py` |

### 产出结构

```
prepared_segments/seg_000001/
├── segment.json                    # 核心控制文件（streams/timeline/quality）
├── data/
│   ├── ego_rgb.mp4                 # H.264 CFR 30fps
│   └── imu.parquet                 # 规范化 IMU（相对时间 + 标准列名）
├── maps/
│   └── rgb_sample_map.parquet      # 输出帧 ↔ 源帧映射（最近邻）
├── calibration/
│   └── calibration.json            # RGB 内参 + 深度/IMU 外参
└── reports/
    └── validation.json             # 6 项自动校验
```

### write-validate 闭环

每次写出后自动验证 6 项：

| 检查项 | 说明 |
|--------|------|
| 文件引用完整性 | segment.json 中引用的所有文件是否真实存在 |
| 视频可解码 | ego_rgb.mp4 能否被 OpenCV 正常打开 |
| 帧数一致 | 视频帧数 == sample_map 行数 |
| 映射时间单调 | output_timestamp_ns 严格递增 |
| IMU 时间单调 | timestamp_ns 非递减 |
| IMU 起点 | 相对时间从 0 附近开始 |

### 配置（config.yaml）

```yaml
video:
  black_threshold: 5
  min_black_duration_s: 0.5

timestamp:
  video_gap_factor: 2.0
  imu_gap_factor: 3.0

output:
  target_fps: 30.0

segment:
  min_duration_s: 1.0
  max_duration_s: 120.0

# 以下为新增参数 (zpds_prepare)
video:
  black_detection:
    mean_intensity_threshold: 5.0
    min_duration_s: 0.5
    edge_tolerance_s: 1.0

timestamp:
  video:
    gap_factor: 2.0
    split_gap_s: 0.5
  imu:
    gap_factor: 3.0
    split_gap_s: 1.0
```

## zpds_prepare/ — 异常检测 → 分段方案（新增）

将"检测到异常"变成"可执行的分段方案"。三个检测器输出统一的 `QualityIssue`，
再由 `segment_planner` 生成候选 Segment。

```
python -m zpds_prepare.main "E:/datasets/egos/墨现" --output output/
```

### 六步管线

| 步骤 | 内容 | 负责模块 |
|------|------|----------|
| ① 读取原始数据 | meta.json / index.jsonl / IMU / MKV 路径 | `readers/guida_reader.py` |
| ② 运行检测器 | 黑屏区间 + 视频缺口 + IMU 缺口 → QualityIssue 列表 | `detectors/` |
| ③ 汇总分析 | 按类型 / 按处置统计 Issue | `decisions/segment_planner.py` |
| ④ 写出质量报告 | quality_issues.json | `writers/quality_writer.py` |
| ⑤ 生成候选 Segment | 根据 trim/split 决策切分 Session | `decisions/segment_planner.py` |
| ⑥ 写出分段方案 | segment_candidates.json | `writers/candidate_writer.py` |

### 统一异常记录格式

所有检测器输出 `QualityIssue` 数据类，不再 `print()`：

```python
@dataclass
class QualityIssue:
    issue_type: str      # continuous_black_frames | timestamp_gap | imu_gap
    stream_id: str       # ego_rgb | ego_imu
    start_ns: int        # 异常起始 (设备时钟)
    end_ns: int          # 异常结束 (设备时钟)
    severity: str        # warning | error | critical
    decision: str        # trim | split | keep_with_flag | quarantine
    details: dict        # 附加信息 (帧数、间隔、阈值等)
```

### 处置决策

| 位置 | 决策 | 说明 |
|------|------|------|
| 开头连续黑屏 | `trim` | 裁剪掉，不计入有效范围 |
| 结尾连续黑屏 | `trim` | 裁剪掉，不计入有效范围 |
| 中间长黑屏 | `split` | 在此处切分 |
| 中间短黑屏 | `keep_with_flag` | 保留并标记 |
| 视频小缺口 (<0.5s) | `keep_with_flag` | 保留并标记 |
| 视频大缺口 (≥0.5s) | `split` | 在此处切分 |
| IMU 小缺口 (<1.0s) | `keep_with_flag` | 保留并标记 |
| IMU 大缺口 (≥1.0s) | `split` | 在此处切分或隔离 |

### 产出结构

```
output/
├── quality_issues.json       # 所有异常的统一记录
└── segment_candidates.json   # 候选 Segment 列表
```

## batch_prepare.py — 批量 Prepared Segment 生成（新增）

读取 segment_candidates.json，对每个候选区间自动生成完整 Prepared Segment。

```
python batch_prepare.py
python batch_prepare.py --candidates output/segment_candidates.json --output prepared_segments/
```

### 工作流程

```
segment_candidates.json
        │
        ▼
  预加载共享资源（index.jsonl + calibration）
        │
        ├── candidate_000001 → seg_000001/
        ├── candidate_000002 → seg_000002/
        └── candidate_000003 → seg_000003/
                    │
                    ▼
          prepared_segments/
          ├── batch_summary.json       ← 批量汇总
          ├── seg_000001/              ← 完整 Prepared Segment
          ├── seg_000002/
          └── seg_000003/
```

### 共享优化

- **index.jsonl** — 全 Session 只读取一次
- **calibration.json** — 全 Session 只提取一次（标定不随 span 变化）
- 每个 Segment 独立转码视频、生成 sample_map、裁剪 IMU、写入验证

### 批量汇总 (batch_summary.json)

```json
{
  "run_at": "2026-07-23T14:41:19",
  "total_segments": 1,
  "pass": 1,
  "fail": 0,
  "total_rgb_frames": 984,
  "total_imu_samples": 1964,
  "segments": [
    {
      "segment_id": "seg_000001",
      "status": "pass",
      "duration_s": 32.77,
      "rgb_frames": 984,
      "imu_samples": 1964,
      "checks": { ... },
      "elapsed_s": 57.3
    }
  ]
}
```

### 与 prepare_segment.py 的区别

| | prepare_segment.py | batch_prepare.py |
|---|---|---|
| 输入 | 硬编码路径 + QC 结果 | segment_candidates.json |
| Span 来源 | `span_determiner.py` 自动判定 | 候选方案文件中指定 |
| Segment 数量 | 始终 1 个 | 按候选方案批量 |
| 共享资源 | 无 | index + calibration 只加载一次 |
| 用途 | 单次开发调试 | 批量生产 |
